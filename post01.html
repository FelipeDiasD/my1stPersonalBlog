GIT e versionamento de código

Versionamento de código e o Git em si acredito serem tópicos essenciais para quem está aos poucos entrando no mundo da programação. Neste post vou explicar o que é git e como funciona o versionamento de código.

Versionamento

O que seria esse tal versionamento? Bem, quando se fala em versionamento, já se pode fazer a ligação com a palavra versão. Não é incomum que você já tenha feito um trabalho de faculdade ou no ensino médio e teve que realizar várias alterações em arquivos em que, na teoria, eram as versões finais, resultando na seguinte situação:

O versionamento vem de encontro com essa situação ao se ter em um sistema, uma forma de se ter todos os registros de alterações realizadas em um determinado arquivo, nos dando a possibilidade de voltar em uma versão anterior, ou adicionar alguma alteração sem ter que necessariamente salvar algo separadamente. Adicionalmente, isso também nos dá a possibilidade de que possamos trabalhar em equipe de maneira mais coesa, ao partirmos de uma mesma versão de determinado arquivo e cada um adicionar uma alteração diferente e assim no final conseguindo unir essas versões em uma só com todas as alterações. No nosso caso, esse conceito é aplicado um arquivo de código.

O que é Git?

Git é um sistema de versionamento de código, que aplica o conceito explicado acima, de forma que o mesmo guarda os registros de versão do código como snapshots (“tira uma foto” de como o arquivo estava no momento em que foi salvo no sistema), salvando também a referência/caminho para o mesmo. Sendo assim, se forma um fluxo de snapshots.

Isso difere o git de outros sistemas de versionamento, pois outros sistemas se utilizam de um conceito de versionamento baseado em arquivos em si, e não em um snapshot. O que gera um fluxo de arquivos e modificações nesse arquivo.

Outro ponto de destaque do Git é o fato de que ele opera localmente. Todas as operações e gravações realizadas pelo git estão localmente no seu disco, fazendo com que a maioria seja quase instantânea. Isso também é um salva vidas quando se está offline, seja num avião ou mesmo se a internet caiu.

Funcionamento do Git

Mas onde o Git guarda esses dados? Bem, para isso o Git cria um repositório, que nada mais é que a base de dados onde estarão todas suas alterações. E para funcionar ele utiliza uma lógica que atribui estados para os arquivos. Tais estados são:

  • Modified: O arquivo em questão foi alterado, mas ainda não foi feito o commit ou seja, registro dessa alteração na base de dados.
  • Staged: As alterações realizadas no arquivo em questão foram marcadas para serem salvas/registradas no próximo commit ou seja, na próxima vez que o registro na base de dados for realizado.
  • Commited: Os dados da alteração realizada no arquivo estão registrados e salvos na base de dados.

Com esses estados em mente, no Git temos três seções principais:

  • Working tree: é basicamente onde realizamos as alterações. Os dados são puxados do repositório (Git repository) e disponibilizados para realizarmos qualquer alteração.
  • Staging area: Aqui estarão os dados marcados para serem registrados/salvos no próximo commit.
  • Git repository: Esta é a parte mais importante do sistema, o seu repositório, aquele que comentamos no qual estão salvos todos os dados e metadados sobre os seus arquivos e projeto como um todo.
